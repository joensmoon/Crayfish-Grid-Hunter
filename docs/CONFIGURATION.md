# 详细配置说明 (Configuration)

Crayfish Grid Hunter 的核心引擎 `grid_hunter_v5.py` 中定义了一个 `UserConfig` 数据类，包含了所有可供用户自定义的参数。在 OpenClaw 中，您无需修改代码，只需在对话中用自然语言提出即可。

---

## 1. 类别 A：次新币横盘类 (Category A)

这些参数用于控制"次新币网格"的筛选逻辑。

| 参数名 | CLI 参数 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- |
| `contract_recent_days` | `--contract-recent-days` | `90` | 合约在币安上线的天数上限。默认 90 天内视为次新币。 |
| `cat_a_mcap_min` | `--cat-a-mcap-min` | `10,000,000` | 次新币市值下限（美元）。默认 1000 万美元。 |
| `cat_a_mcap_max` | `--cat-a-mcap-max` | `200,000,000` | 次新币市值上限（美元）。默认 2 亿美元。 |
| `volume_shrink_ratio` | `--volume-shrink-ratio` | `0.50` | 成交量萎缩比例。24h 成交量必须小于 7 天平均成交量的 50%。 |
| `atr_sideways_pct` | `--atr-sideways-pct` | `2.0` | 真实波动幅度（ATR14）占比。小于 2% 视为横盘。 |
| `bb_width_sideways` | `--bb-width-sideways` | `5.0` | 布林带宽度占比。小于 5% 视为极窄幅震荡。 |
| `adx_sideways` | `--adx-sideways` | `20.0` | 平均趋向指数（ADX14）。小于 20 视为无明显趋势。 |

**自然语言示例**：
> "次新币网格，合约上线天数改为 120 天，市值范围 500 万到 1 亿"

## 2. 类别 B：高波动套利类 (Category B)

这些参数用于控制"高波动套利"的筛选逻辑。

| 参数名 | CLI 参数 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- |
| `mcap_min` | `--mcap-min` | `200,000,000` | 最小市值（美元）。默认 2 亿美元。 |
| `mcap_max` | `--mcap-max` | `1,000,000,000` | 最大市值（美元）。默认 10 亿美元。 |
| `high_vol_48h_min_pct` | `--high-vol-48h-min-pct` | `10.0` | 48 小时涨跌幅绝对值下限（%）。默认 10%。 |
| `turnover_min` | `--turnover-min` | `0.50` | 24h 换手率下限。默认 50%（即成交量达到市值的一半）。 |
| `rv_min` | `--rv-min` | `15.0` | 年化实现波动率下限（%）。默认 15%。 |
| `min_quote_volume` | — | `10,000,000` | 最小 24h USDT 交易量，确保流动性。 |

**自然语言示例**：
> "高波动套利，市值范围放宽到 1 亿到 50 亿，48h 涨跌幅改为 15%，换手率降低到 30%"

## 3. 网格参数 (Grid Parameters)

这些参数控制生成的等比网格策略的形态。

| 参数名 | CLI 参数 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- |
| `leverage` | `--leverage` | `5` | 合约杠杆倍数。建议不超过 20x。 |
| `min_grid_profit` | `--min-grid-profit` | `0.008` | 扣除手续费后的单格最低利润率（0.8%）。低于此值时系统自动减少网格数。 |
| `max_grid_profit` | — | `0.012` | 单格最高利润率目标（1.2%）。 |
| `stop_loss_pct` | `--stop-loss-pct` | `5.0` | 硬止损位。设置在网格下轨下方 5% 处。 |
| `max_exposure` | — | `0.08` | 最大账户风险敞口。建议单网格投入不超过总资金的 8%。 |

**自然语言示例**：
> "网格猎手，杠杆改为 3 倍，止损设为 8%，单格利润最低 1.5%"

## 4. 扫描控制与回测参数

| 参数名 | CLI 参数 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- |
| `top_n` | `--top-n` | `3` | 每个分类返回的推荐标的数量。 |
| `max_symbols` | `--max-symbols` | `200` | 扫描全市场时分析的合约数量上限（按交易量排序）。 |
| `backtest_days` | — | `30` | 历史回测的天数。 |
| `backtest_interval` | — | `"1h"` | 回测使用的 K 线级别。 |
| — | `--no-backtest` | `false` | 跳过历史回测步骤（加快速度）。 |
| — | `--no-progress` | `false` | 不显示进度条（适合日志输出）。 |
| — | `--mode` | `all` | 筛选模式：`all`=双分类全量, `cat-a`=仅次新币, `cat-b`=仅高波动。 |

**自然语言示例**：
> "帮我跑一次网格猎手，只看前 5 个结果，扫描前 100 个币种，跳过回测"
